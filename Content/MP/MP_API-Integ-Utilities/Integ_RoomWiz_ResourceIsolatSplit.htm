Resource Isolation - Splitting Application Pools

Background 

The RoomWizard Connector is setup as a web application, side-by-side with the MeetingPlanner and MeetingPlanner_WS applications.  All of these applications, by default, share an application pool.  If issues are encountered or a better allocation of resources on the application server is required or desired, the RoomWizard Connector can be split into its own application pool.  This will allow the MeetingPlanner users and RoomWizard Connector to maintain separate connection pools and memory allocations.

Implementation

  1. Open IIS.
  2. Create a new application pool.  this new application pool will be the RoomWizard App Pool and should mirror the MeetingPlanner App Pool setup (see below). 

Image_001.png

Image_002.png

  1. Once the RoomWizard App Pool is created, select the Default Web Site > RoomWizard application.  
  2. Click "Advanced Settings" and update the Application Pool from MeetingPlannerAppPool to the RoomWizard pool that was just created.
  3. After completion, it may also be beneficial to reset IIS.

SSRS Reporting

Create Data Source

Background 

The SSRS reporting engine will require a data source be setup so that a connection can be established to the MeetingPlanner database.  Once this data source is available, all reports can be linked to the one data source.  This way, if the SQL server information ever changes, you will be able to update the connections for all reports by simply updating this data source.

Steps

  1. Browse to your SSRS instance.
  2. Click "New Data Source."

Image_018.png

  1. Create the new data source.
  • Select "Hide in tile view"
  • Update the Connection String to match your SQL server and database names
  1. Click Apply
  2. Update the Security "tab" as necessary (this will typically be more limited than the users that have access to reports)

Introduction

Introduction to MeetingPlanner SSRS Reporting

Report Manager is a Web-based report access and management tool used to:

  • View, search, print, and subscribe to reports
  • Create, secure, and maintain reports on the server
  • Configure role-based security
  • Configure report execution properties, and report parameters
  • Create report models
  • Create shared schedules and shared data sources to make schedules and data source connections more manageable
  • Create data-driven subscriptions that roll out reports to a recipient list
  • Create linked reports to reuse and repurpose an existing report in different ways

A complete introduction to SSRS Report Manager can be found at: https://msdn.microsoft.com/en-us/library/ms157147(v=sql.110).aspx.

A complete, up-to-date guide of Reporting Services Features can be found at: https://msdn.microsoft.com/en-us/library/ms159273(v=sql.105).aspx.

Upload a Report

Background 

Emergingsoft will be posting various reports that we think may be helpful to our customers, and will provide assistance updating or customizing reports.  If you choose to download a report or receive a .rdl file from our support team, please see the instructions below for how to upload the report to your SSRS instance.

Steps 

  1. Browse to your SSRS instance.
  2. Click "Upload File."

Image_002.png

  1. Upload the report (.rdl).
  2. From the dropdown menu for the report, click "Manage."

Image_003.png

  1. Click "Data Sources" > "A shared data source"
  2. Browse to your current MeetingPlanner data source.

*In the event that you cannot browse to your shared data source, you can manually type in the path.  The default path would be "/MeetingPlanner" (no quotes).

  1. Click "Apply."
